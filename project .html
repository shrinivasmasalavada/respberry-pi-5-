02Hardware Setup

Hardware Connections — Step by Step

Follow these steps for both Raspberry Pi boards. The SX126X HAT plugs directly onto the GPIO header — no loose wires needed.

1
Power off your Raspberry Pi completely

Unplug the USB-C power cable. Never attach a HAT while the board is on — this can short the pins and destroy both the Pi and the HAT.

2
Attach the antenna to the HAT

Screw or push the antenna connector onto the SMA port on the LoRa HAT. Make sure it is tight. A loose antenna causes poor range or damage.

3
Align the HAT over the 40-pin GPIO header

The HAT has a matching 40-pin female socket on its underside. Line up pin 1 (the corner near the microSD slot) carefully. Every pin must align — check with a torch before pressing down.

4
Press the HAT down firmly and evenly

Apply gentle, even pressure. You will hear/feel a soft click when it is fully seated. Do not force it — if it resists, realign and try again.

5
Check the jumper settings on the HAT

Most SX126X HATs have jumpers to select the SPI interface and reset pins. For Raspberry Pi, set them to SPI0 mode. Check your HAT's datasheet if unsure — Waveshare HATs default to SPI0.

6
Power the Pi back on

Reconnect the USB-C power. The HAT's LED (if present) may blink briefly during startup. Repeat all steps for your second Raspberry Pi.

05Setup

Install Required Libraries

Run these commands on both Raspberry Pi boards. Open a terminal (press Ctrl+Alt+T or use the Applications menu).

Step 1 — Enable SPI. The SX126X HAT talks over SPI. You must turn it on first:

🖥 Terminal 
$ sudo raspi-config
# Navigate to: Interface Options → SPI → Enable → Yes → Finish
$ sudo reboot

Step 2 — Update the system and install Python tools:

🖥 Terminal 
$ sudo apt update && sudo apt upgrade -y
$ sudo apt install python3-pip python3-dev -y

Step 3 — Install the Python libraries for SPI and GPIO:

🖥 Terminal 
$ pip3 install spidev RPi.GPIO

Step 4 — Verify SPI is active after reboot:

🖥 Terminal 
$ ls /dev/spidev*
/dev/spidev0.0  /dev/spidev0.1
# If you see these two files, SPI is working correctly!
💡
If /dev/spidev* shows nothing, SPI is not enabled. Go back to raspi-config and enable it again, then reboot.

📄 sender.py — run on Pi A

#!/usr/bin/env python3
"""
sender.py  —  Run this on Raspberry Pi A
Sends a text message over LoRa radio to Pi B
"""

import time
import spidev
import RPi.GPIO as GPIO
from lora_module import SX126X

# ── GPIO pin numbers (BCM numbering) ──────────────────────
RESET_PIN = 18   # Physical pin 12  — HAT RESET
BUSY_PIN  = 20   # Physical pin 38  — HAT BUSY
DIO1_PIN  = 16   # Physical pin 36  — HAT DIO1/IRQ
CS_PIN    = 8    # Physical pin 24  — SPI0 CE0

# ── LoRa radio settings ────────────────────────────────────
# IMPORTANT: Both Pi A and Pi B must use IDENTICAL settings!
FREQUENCY   = 868.0  # MHz — use 915.0 for USA/Canada
BANDWIDTH   = 125    # kHz — wider = faster but less range
SPREAD_FACTOR = 7    # 7–12 — higher = longer range but slower
CODING_RATE   = 5    # 5–8  — error correction rate
TX_POWER      = 14   # dBm  — transmit power (max ~22 for SX1268)

def main():
    print("===================================")
    print("  LoRa Sender — Raspberry Pi A")
    print("===================================")

    # Set up GPIO
    GPIO.setmode(GPIO.BCM)
    GPIO.setwarnings(False)

    # Create LoRa radio object
    lora = SX126X(
        spi_bus=0,
        spi_device=0,
        cs_pin=CS_PIN,
        reset_pin=RESET_PIN,
        busy_pin=BUSY_PIN,
        dio1_pin=DIO1_PIN
    )

    # Initialise the module
    lora.begin()
    lora.set_frequency(FREQUENCY)
    lora.set_bandwidth(BANDWIDTH)
    lora.set_spreading_factor(SPREAD_FACTOR)
    lora.set_coding_rate(CODING_RATE)
    lora.set_tx_power(TX_POWER)

    print(f"[OK] LoRa ready at {FREQUENCY} MHz")
    print("[INFO] Sending messages every 5 seconds...")
    print("[INFO] Press Ctrl+C to stop.\n")

    message_count = 1

    try:
        while True:
            # Build the message string
            message = f"Hello from Pi A! Message #{message_count}"

            print(f"[TX] Sending: '{message}'")

            # Encode to bytes and transmit
            lora.transmit(message.encode('utf-8'))

            print(f"[TX] Sent! Waiting 5 seconds...\n")

            message_count += 1
            time.sleep(5)

    except KeyboardInterrupt:
        print("\n[STOP] Sender stopped by user.")

    finally:
        lora.sleep()
        GPIO.cleanup()
        print("[OK] GPIO cleaned up.")

if __name__ == "__main__":
    main()
📄 receiver.py — run on Pi B
Python 3 
#!/usr/bin/env python3
"""
receiver.py  —  Run this on Raspberry Pi B
Listens for LoRa messages and prints them on screen
"""

import time
import spidev
import RPi.GPIO as GPIO
from lora_module import SX126X

# ── GPIO pin numbers (same as sender!) ───────────────────
RESET_PIN = 18
BUSY_PIN  = 20
DIO1_PIN  = 16
CS_PIN    = 8

# ── LoRa settings (MUST match sender.py exactly!) ────────
FREQUENCY     = 868.0
BANDWIDTH     = 125
SPREAD_FACTOR = 7
CODING_RATE   = 5

def on_receive(rssi, snr, data):
    """Called automatically when a packet arrives."""
    timestamp = time.strftime("%H:%M:%S")
    try:
        message = data.decode('utf-8')
    except:
        message = f"[binary data: {data.hex()}]"

    print(f"\n┌─────────────────────────────────")
    print(f"│ 📨 MESSAGE RECEIVED at {timestamp}")
    print(f"│ Content : {message}")
    print(f"│ RSSI    : {rssi} dBm  (signal strength)")
    print(f"│ SNR     : {snr} dB   (signal quality)")
    print(f"└─────────────────────────────────")

def main():
    print("===================================")
    print("  LoRa Receiver — Raspberry Pi B")
    print("===================================")

    # Set up GPIO
    GPIO.setmode(GPIO.BCM)
    GPIO.setwarnings(False)

    # Create LoRa radio object
    lora = SX126X(
        spi_bus=0,
        spi_device=0,
        cs_pin=CS_PIN,
        reset_pin=RESET_PIN,
        busy_pin=BUSY_PIN,
        dio1_pin=DIO1_PIN
    )

    # Initialise — settings MUST match sender!
    lora.begin()
    lora.set_frequency(FREQUENCY)
    lora.set_bandwidth(BANDWIDTH)
    lora.set_spreading_factor(SPREAD_FACTOR)
    lora.set_coding_rate(CODING_RATE)

    # Register our callback function
    lora.on_receive = on_receive

    # Put the radio into continuous receive mode
    lora.receive()

    print(f"[OK] Listening on {FREQUENCY} MHz...")
    print("[INFO] Waiting for messages from Pi A...")
    print("[INFO] Press Ctrl+C to stop.\n")

    try:
        while True:
            # Poll for incoming packets every 100 ms
            lora.poll()
            time.sleep(0.1)

    except KeyboardInterrupt:
        print("\n[STOP] Receiver stopped by user.")

    finally:
        lora.sleep()
        GPIO.cleanup()
        print("[OK] GPIO cleaned up.")

This helper module wraps the low-level SPI commands into simple Python functions. Save it as lora_module.py in the same folder as both sender.py and receiver.py — on both Pi boards.

📄 lora_module.py — copy to BOTH boards
Python 3 
#!/usr/bin/env python3
"""
lora_module.py — SX126X driver for Raspberry Pi
Provides simple send/receive wrappers over SPI.
Copy this file to BOTH Pi A and Pi B.
"""

import time
import spidev
import RPi.GPIO as GPIO

# ── SX126X SPI command codes ──────────────────────────────
_CMD_SET_SLEEP          = 0x84
_CMD_SET_STANDBY        = 0x80
_CMD_SET_PACKET_TYPE    = 0x8A
_CMD_SET_RF_FREQUENCY   = 0x86
_CMD_SET_TX_PARAMS      = 0x8E
_CMD_SET_MODULATION     = 0x8B
_CMD_SET_PACKET_PARAMS  = 0x8C
_CMD_SET_TX             = 0x83
_CMD_SET_RX             = 0x82
_CMD_WRITE_BUFFER       = 0x0E
_CMD_READ_BUFFER        = 0x1E
_CMD_GET_IRQ_STATUS     = 0x12
_CMD_CLEAR_IRQ          = 0x02
_CMD_GET_RX_BUFFER_STATUS = 0x13
_CMD_GET_PACKET_STATUS  = 0x14
_CMD_SET_DIO_IRQ        = 0x08
_CMD_CALIBRATE          = 0x89

_XTAL_FREQ = 32e6   # 32 MHz crystal
_PREAMBLE_LEN = 12  # symbols


class SX126X:
    """
    Minimal SX126X (SX1261/SX1262/SX1268) driver.
    Supports LoRa packet mode with simple TX/RX.
    """

    def __init__(self, spi_bus, spi_device, cs_pin,
                  reset_pin, busy_pin, dio1_pin):
        self.spi_bus    = spi_bus
        self.spi_device = spi_device
        self.cs_pin     = cs_pin
        self.reset_pin  = reset_pin
        self.busy_pin   = busy_pin
        self.dio1_pin   = dio1_pin
        self.on_receive = None   # Set this to a callback function
        self._spi       = None
        self._rx_buffer_size = 0
        self._rx_buffer_offset = 0

    # ── Initialisation ─────────────────────────────────────
    def begin(self):
        """Set up SPI, GPIO, and put the module in standby."""
        GPIO.setup(self.reset_pin, GPIO.OUT)
        GPIO.setup(self.busy_pin,  GPIO.IN)
        GPIO.setup(self.dio1_pin,  GPIO.IN)

        # Hard reset the module
        GPIO.output(self.reset_pin, GPIO.LOW)
        time.sleep(0.01)
        GPIO.output(self.reset_pin, GPIO.HIGH)
        time.sleep(0.1)
        self._wait_busy()

        # Open SPI bus
        self._spi = spidev.SpiDev()
        self._spi.open(self.spi_bus, self.spi_device)
        self._spi.max_speed_hz = 2_000_000  # 2 MHz SPI clock
        self._spi.mode = 0

        # Standby mode, then select LoRa packet type
        self._cmd([_CMD_SET_STANDBY, 0x00])  # STDBY_RC
        self._cmd([_CMD_SET_PACKET_TYPE, 0x01])  # LoRa mode
        self._cmd([_CMD_CALIBRATE, 0x7F])    # Calibrate all
        time.sleep(0.01)
        self._wait_busy()

    # ── Configuration methods ──────────────────────────────
    def set_frequency(self, freq_mhz):
        """Set LoRa carrier frequency in MHz."""
        frf = int((freq_mhz * 1e6 / _XTAL_FREQ) * (1 << 25))
        self._cmd([
            _CMD_SET_RF_FREQUENCY,
            (frf >> 24) & 0xFF,
            (frf >> 16) & 0xFF,
            (frf >>  8) & 0xFF,
             frf        & 0xFF,
        ])

    def set_tx_power(self, dbm):
        """Set transmit power in dBm (2..22 for SX1268)."""
        dbm = max(2, min(22, dbm))
        self._cmd([_CMD_SET_TX_PARAMS, dbm, 0x04])  # ramp 200 µs

    def set_bandwidth(self, bw_khz):
        """Set LoRa bandwidth. Options: 125, 250, 500 kHz."""
        bw_map = {125: 4, 250: 5, 500: 6}
        self._bw = bw_map.get(bw_khz, 4)

    def set_spreading_factor(self, sf):
        """Set spreading factor (7–12). Higher = longer range."""
        self._sf = max(5, min(12, sf))

    def set_coding_rate(self, cr):
        """Set coding rate denominator (5–8)."""
        self._cr = max(5, min(8, cr)) - 4

    def _apply_modulation(self):
        """Push modulation settings to the chip."""
        ldro = 1 if self._sf >= 11 else 0
        self._cmd([_CMD_SET_MODULATION,
                   self._sf, self._bw, self._cr, ldro])

    # ── TX — transmit one packet ───────────────────────────
    def transmit(self, data: bytes):
        """Send a bytes payload. Blocks until TX completes."""
        self._cmd([_CMD_SET_STANDBY, 0x00])
        self._apply_modulation()

        payload_len = len(data)
        self._cmd([_CMD_SET_PACKET_PARAMS,
                   (_PREAMBLE_LEN >> 8) & 0xFF,
                    _PREAMBLE_LEN        & 0xFF,
                   0x00,          # explicit header
                   payload_len,
                   0x01,          # CRC on
                   0x00])         # IQ standard

        # Write payload to chip FIFO
        self._cmd([_CMD_WRITE_BUFFER, 0x00] + list(data))
        self._wait_busy()

        # Trigger transmit (timeout ~5 s)
        self._cmd([_CMD_SET_TX, 0x09, 0xC4, 0x00])
        timeout = time.time() + 6
        while time.time() < timeout:
            irq = self._get_irq()
            if irq & 0x01:  # TxDone
                self._cmd([_CMD_CLEAR_IRQ, 0xFF, 0xFF])
                return True
            time.sleep(0.01)
        raise TimeoutError("LoRa TX timeout — no TxDone IRQ")

    # ── RX — continuous receive mode ──────────────────────
    def receive(self):
        """Start continuous receive mode."""
        self._cmd([_CMD_SET_STANDBY, 0x00])
        self._apply_modulation()
        self._cmd([_CMD_SET_PACKET_PARAMS,
                   0x00, 0x0C,  # preamble 12
                   0x00,          # explicit header
                   0xFF,          # max payload 255 bytes
                   0x01,          # CRC on
                   0x00])
        # Enable RxDone + CRC error IRQs on DIO1
        self._cmd([_CMD_SET_DIO_IRQ,
                   0xFF, 0xFF,  # all IRQs enabled
                   0x02, 0x02,  # DIO1 → RxDone
                   0x00, 0x00,  # DIO2 off
                   0x00, 0x00]) # DIO3 off
        # Continuous RX (no timeout)
        self._cmd([_CMD_SET_RX, 0xFF, 0xFF, 0xFF])

    def poll(self):
        """
        Call this in a loop. Checks for incoming packets.
        Fires on_receive(rssi, snr, data) when one arrives.
        """
        irq = self._get_irq()
        if irq & 0x02:  # RxDone
            self._cmd([_CMD_CLEAR_IRQ, 0xFF, 0xFF])
            # Read how many bytes arrived and where in the FIFO
            status = self._cmd([_CMD_GET_RX_BUFFER_STATUS, 0, 0])
            rx_len    = status[1]
            rx_offset = status[2]
            # Read the payload bytes
            payload = self._cmd([_CMD_READ_BUFFER, rx_offset, 0]
                                + [0] * rx_len)[3:]
            # Get RSSI and SNR
            pkt = self._cmd([_CMD_GET_PACKET_STATUS, 0, 0, 0])
            rssi = -pkt[1] // 2
            snr  =  pkt[2] // 4
            if self.on_receive:
                self.on_receive(rssi, snr, bytes(payload))
        elif irq & 0x40:  # CRC error
            self._cmd([_CMD_CLEAR_IRQ, 0xFF, 0xFF])
            print("[WARN] CRC error — packet corrupted.")

    # ── Sleep ─────────────────────────────────────────────
    def sleep(self):
        self._cmd([_CMD_SET_SLEEP, 0x04])

    # ── Low-level helpers ──────────────────────────────────
    def _wait_busy(self, timeout=3.0):
        """Block until the BUSY pin goes low."""
        deadline = time.time() + timeout
        while GPIO.input(self.busy_pin):
            if time.time() > deadline:
                raise TimeoutError("SX126X BUSY pin stuck high!")
            time.sleep(0.001)

    def _cmd(self, data):
        """Send bytes over SPI, return the response."""
        self._wait_busy()
        resp = self._spi.xfer2(data)
        return resp

    def _get_irq(self):
        """Read the 16-bit IRQ status register."""
        r = self._cmd([_CMD_GET_IRQ_STATUS, 0, 0])
        return (r[1] << 8) | r[2]

07Testing

How to Test the Communication

You need two separate terminal windows — one for each Raspberry Pi. You can SSH into them from your laptop, or connect a keyboard and monitor to each.

1
Open a terminal on Pi B (Receiver) first

Always start the receiver before the sender. This ensures Pi B is listening when Pi A begins transmitting.

2
Navigate to your project folder on Pi B
pi@raspberry-B: ~/lora_project
pi@raspberry-B:~$ cd ~/lora_project
pi@raspberry-B:~/lora_project$ python3 receiver.py
3
Open a terminal on Pi A (Sender) and start sending
pi@raspberry-A: ~/lora_project
pi@raspberry-A:~$ cd ~/lora_project
pi@raspberry-A:~/lora_project$ python3 sender.py
4
Watch the messages appear on Pi B's screen

Every 5 seconds, a new message should appear in Pi B's terminal. RSSI shows signal strength (−50 dBm is excellent, −120 dBm is very weak).

5
Stop both programs with Ctrl+C

Both scripts will clean up GPIO and put the LoRa module to sleep safely.

⚠️
Frequency law! In Europe use 868 MHz. In the USA/Canada use 915 MHz. Using the wrong frequency is illegal in some countries. Check your country's radio regulations before transmitting outdoors.
